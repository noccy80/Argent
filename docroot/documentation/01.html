Global Variables

The core Argent server exposes a few global variables to make programming modules easier. In no particular order, here they are, each indentation representing parenthood of the less indented item:

  • sys
    • logger
      • stdout(message) - Writes `message` to standard output preceded by a timestamp.
      • stderr(message) - Writes `message` to standard error preceded by a timestamp.
      • now() - Returns the current date and time in string format.
    • http
      • http - The node.js http library.
      • url - The node.js url library.
    • fs - The node.js fs library.
    • path - The node.js path library.
    • handlers - An array of all handlers loaded into Argent.
    • config - An object containing server-level configuration options (no module configs should be stored here).
      • server_root - The directory where the server's executables can be found
      • server_url - The preferred URL for use with this Argent server
    • respond(response, httpcode, mimetype, body, headers) - Sends a response to the client connection to whom 'response' refers to. The response uses the code in 'httpcode' (like 200 for success, or 404 for not found), and sends the 'body' as a file of type 'mimetype'. The HTTP headers will be set to the contents of the 'headers' object.
  • mods - An object containing all modules loaded into Argent. For example, mods.session is the session module.

In other words, if you want to log to stderr, you'd run sys.logger.stderr("Error");
