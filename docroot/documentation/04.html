Argent Documentation

httpd.conf

This page concerns the primary Argent configuration file, which is fairly well commented itself, at least in its default form. This file is httpd.conf, and it can be found in the conf directory of your Argent instance.

Argent reads this file line by line and executing each directive in sequence. If there are conflicts between two lines, the one that appears later will always take precedence, and this can be an important distinction to make sometimes.

Aside from comments, the following directives are understood right now:

  • default_file=path/to/file.ext - If Argent receives a request to serve a directory, it will look for this file in that directory to server. A good default here is index.html
  • default_mime_type=mime/type - If Argent cannot determine the mime type of a file, tell the client that it's this one. Good defaults here might be text/html or text/plain
  • document_root=/path/to/doc/root - Argent will only serve files from this directory and its subdirectories.
  • listen_port=number - Argent will listen on this port for incoming HTTP connections. If you want to run Argent on port 80 or 443 (or any port lower than 1024), any reasonable operating system will require you to have some kind of root or otherwise elevated access. By default, Argent listens on port 12321.
  • module_dir=/path/to/mod/dir - All modules and handlers can be found in this directory.
  • mod=module_name - Load the module found in the file module_name.js in the module_dir directory.
  • handler=handler_file,module_it_handles - Loads the handler found in handler_file.js in the module_dir directory, and uses it to handle the already-loaded module called module_it_handles.
  • var=variable.to.set=value - Manually sets the variable variable.to.set to value.
  • ext=list,of,file,extensions,mime/type - Associates the listed file extensions to the given mime type. All files with those extensions that Argents serves will be accompanied by the given mime type.
  • binary=li/st,o/f,mime/types - Tells Argent to serve all of the listed mime types as binary data instead of text data. Mime types that you should list here include all image formats and fonts if you serve those, as well as any binary files, compressed files, etc. that your Argent server serves.
  • error_document,error_number,filename - Tells Argent that if error_number gets returned to the client, it should pass the file at filename (relative to the document root) as the body. For example: error_document,404,errors/err404.html

Other modules may use their own config files, but you will have to read their documentation for that information.

Additionally, comments should be on their own lines (no tacking them on to the end of directives) and should begin with a hash (#) or a semicolon (;). Hashes traditionally imply commentary while semicolons traditionally imply default values for optional directives.
